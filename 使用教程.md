**API 地址：** `http://ecs.wwfeng3045.top:5000`

## 🛠 安装部署

### 环境要求
- Linux 服务器（Ubuntu/CentOS 等）
- Python 3.8+
- poppler-utils（用于 PDF 处理）

### 步骤 1：安装依赖

```bash
# Ubuntu/Debian
apt-get update && apt-get install -y poppler-utils python3-venv

# CentOS/RHEL
yum install -y poppler-utils python3
```

### 步骤 2：上传代码到服务器

```bash
# 创建目录
mkdir -p /opt/feishu-pdf-signature

# 上传代码（使用 scp 或 rsync）
scp -r ./feishu-pdf-signature/* root@your-server:/opt/feishu-pdf-signature/
```

### 步骤 3：创建 Python 虚拟环境

```bash
cd /opt/feishu-pdf-signature
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

### 步骤 4：配置参数

编辑 `config.json`：

```json
{
  "app_token": "你的app_token",
  "table_id": "你的table_id",
  "pdf_field": "原始PDF",
  "signed_pdf_field": "已签字PDF",
  "signer_field": "签字人",
  "feishu_app_id": "飞书应用ID",
  "feishu_app_secret": "飞书应用密钥"
}
```

### 步骤 5：配置开机启动服务

创建 systemd 服务文件：

```ini
[Unit]
Description=Feishu PDF Signature Service
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/feishu-pdf-signature
Environment="PATH=/opt/feishu-pdf-signature/venv/bin"
ExecStart=/opt/feishu-pdf-signature/venv/bin/python server.py
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

安装服务：

```bash
# 复制服务文件
cp feishu-sign.service /etc/systemd/system/

# 重载 systemd
systemctl daemon-reload

# 启用开机启动
systemctl enable feishu-sign

# 启动服务
systemctl start feishu-sign

# 查看状态
systemctl status feishu-sign
```

### 步骤 6：开放防火墙端口

```bash
# Ubuntu/Debian (ufw)
ufw allow 5000/tcp

# CentOS (firewalld)
firewall-cmd --permanent --add-port=5000/tcp
firewall-cmd --reload
```

## 🚀 快速开始

### 1. 在多维表格中准备数据

创建一个多维表格，包含以下字段：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 文档名称 | 文本 | 文件名称 |
| 原始PDF | 附件 | 待签字的 PDF 文件 |
| 签字链接 | 文本 | 自动生成的签字链接（API 回填） |
| 已签字PDF | 附件 | 签字完成后的 PDF（自动回填） |

### 2. 配置飞书自动化

在多维表格中创建自动化流程：

**触发条件：** 当记录满足条件时
- 例如：「签字链接」为空时触发

**执行操作：** 发送 HTTP 请求
- **方法：** `POST`
- **URL：** `http://ecs.wwfeng3045.top:5000/api/generate_link`
- **Header：** `Content-Type: application/json`
- **Body：**
```json
{
  "record_id": "{{record_id}}",
  "app_token": "你的app_token",
  "table_id": "你的table_id",
  "pdf_field": "原始PDF",
  "sign_link_field": "签字链接",
  "signed_pdf_field": "已签字PDF",
  "personal_base_token": "你的personal_base_token"
}
```

### 3. 获取签字链接

自动化触发后，API 会：
1. 从「原始PDF」字段下载 PDF
2. 生成签字链接
3. 自动回填到「签字链接」字段

### 4. 发起签字

打开多维表格，点击「签字链接」字段中的链接，打开签字页面。

### 5. 签字操作

#### 5.1 选择签字位置
- 在左侧 PDF 预览区点击，选择签字位置
- 会出现一个虚线框标注签字区域

#### 5.2 调整签字框大小
- 拖动**框内区域**移动位置
- 拖动**边角把手**缩放大小（8个缩放点）
- 拖动**边框把手**单向拉伸

#### 5.3 手写签字
- 在右侧签字板上手写签名

#### 5.4 手机扫码签字（推荐）
- 点击右下角 📱 按钮
- 扫描二维码在手机端签字
- 手机上手写更方便

#### 5.5 提交签字
- 点击「提交签字」按钮
- 系统会将签字嵌入 PDF

### 6. 完成

签字完成后：
- 签字后的 PDF 自动回填到「已签字PDF」字段
- 可以下载查看

## 📡 API 参数说明

| 参数 | 必填 | 说明 |
|------|------|------|
| `record_id` | ✅ | 多维表格记录 ID |
| `app_token` | ❌ | 多维表格 app_token（不填则用配置文件） |
| `table_id` | ❌ | 数据表 ID（不填则用配置文件） |
| `pdf_field` | ❌ | PDF 原始文件字段名，默认「原始PDF」 |
| `signed_pdf_field` | ❌ | 签字完成 PDF 回填字段，默认「已签字PDF」 |
| `sign_link_field` | ❌ | 签字链接回填字段 |
| `personal_base_token` | ❌ | 多维表格授权码（多表格场景必须） |

---

## 🔧 获取参数的方法

### record_id
在多维表格打开记录详情，URL 中可找到 `record_id`。

### app_token
多维表格 URL：`https://my.feishu.cn/base/xxx`
`xxx` 部分就是 app_token。

### table_id
多维表格 URL：`https://my.feishu.cn/base/xxx?table=yyy`
`yyy` 部分就是 table_id。

### personal_base_token
1. 打开多维表格
2. 右上角「···」→「设置」→「授权码」
3. 生成或复制授权码

## 🆕 新功能：多表格支持

每个多维表格可以有不同的配置：

```json
{
  "record_id": "recxxxxx",
  "app_token": "不同表格的app_token",
  "table_id": "不同表格的table_id",
  "pdf_field": "原始PDF",
  "sign_link_field": "签字链接",
  "signed_pdf_field": "已签字PDF",
  "personal_base_token": "该表格的授权码"
}
```

每次签字的配置独立存储，支持多个表格、多个签字流程。

## ❓ 常见问题

### Q: 签字提交后显示成功，但下载的 PDF 是空白/未签字？
A: 检查云服务器是否安装了 poppler：`apt-get install poppler-utils`

### Q: personal_base_token 无效？
A: 需要在多维表格中重新生成授权码

### Q: 字段名填写错误？
A: API 返回错误时会提示「FieldNameNotFound」，请检查字段名是否完全匹配

### Q: 服务无法启动？
A: 检查服务状态：`systemctl status feishu-sign`
   查看日志：`journalctl -u feishu-sign -f`
